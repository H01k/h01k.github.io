def ctor_heap_head( size, flags, prev_size, remain_size):
    chksum =  ((size >> 4) & 0xff) ^ ((size >> 4) >> 8) ^ flags
    val = (size >> 4) | (flags << 16) | (chksum << 24) | (prev_size >> 4 << 32) | (remain_size << 56)

LFH 分析

其实就是每个大小会有一块内存来进行分配 每次分配的位置是随机的，攻击LFH最重要的就是获取确定位置的堆块
